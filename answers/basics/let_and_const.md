1. Если идет обращение к переменной до ее обьявления через var, то она будет доступна и равна undefined, потому что всплывает. Но всплытие переменной будет только в рамках той функции, где она была создана (в противном случае будет ошибка).
2. Переменная, обьявленная через var, имеет функциональную область видимости, а через let/const - блочную. Вот хороший пример.
```js
for (var i = 0; i < 10; i++) {
    setTimeout(() => {
        console.log(i); 
    }, 1000); 
}; 
```
В данном случае '10' 10 раз, потому что var игнорирует скобки, и переменная i существует лишь 1 раз - и будет равна 10 в момент выполнения тасок (вспомни, что i через var будет доступна даже вне цикла). А при let на каждой итерации будет своя новая переменная, которая будет браться из замыкания. Задачу можно исправить так:
```js
for (var i = 0; i < 10; i++) {
    setTimeout(function(index) {
        return function() {
        console.log(index);
        };
    }(i), 1000);
};
```
